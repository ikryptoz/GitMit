GitMit Security and Privacy

GitMit is designed to keep message content encrypted end-to-end while being transparent about security boundaries and limitations.

E2EE message content Fingerprint verification Multi-device key continuity Public security docs

What Is Protected

  • Message plaintext is encrypted on client devices.
  • Server-side storage is intended to hold encrypted payloads, not message plaintext.
  • Fingerprint checks support anti-MITM verification between users.

What Is Not Fully Hidden

  • Operational metadata exists for routing and synchronization.
  • Public repositories can be cloned.
  • Web client code is downloadable by browsers by design.

Security here means layered controls and transparency, not impossible-to-copy client code.

Recommended User Actions

  • Verify fingerprints for sensitive chats.
  • Use trusted devices and screen lock.
  • Avoid shared/untrusted web sessions.
  • Report suspicious identity changes.

Maintainer Security Priorities

  • Never commit secrets or signing passwords.
  • Keep authz checks server-side.
  • Rotate tokens/keys after incidents.
  • Use legal/trademark controls against impersonation.
